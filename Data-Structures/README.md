## 🎯 前端算法特点

### 与后端算法的区别
1. **复杂度要求**：前端更注重实现，而不是最优解
2. **数据结构**：更多使用数组和对象，较少使用复杂数据结构
3. **实用导向**：更多与实际业务场景结合
4. **代码简洁**：更注重代码可读性和易维护性

### 重点掌握
1. **数组操作**：熟记数组常用方法的时间复杂度
2. **字符串处理**：掌握正则表达式和字符串方法
3. **递归思想**：理解递归和迭代的转化
4. **复杂度分析**：能够分析算法的时间和空间复杂度

## 📝 解题技巧

### 通用解题步骤
1. **审题**：明确输入输出，边界条件
2. **举例**：用简单例子验证理解
3. **思路**：先想思路，再写代码
4. **实现**：代码实现，考虑边界
5. **测试**：用不同用例测试
6. **优化**：时间和空间优化

### 常见解题模式
1. **双指针**：左右指针、快慢指针
2. **滑动窗口**：固定大小或可变窗口
3. **哈希表**：空间换时间
4. **递归/回溯**：树形结构、组合问题
5. **动态规划**：最优子结构、重叠子问题

## ⏱️ 复杂度分析速查

### 时间复杂度
Included in /root/.openclaw/workspace/frontend-interview-top50/Data-Structures/README.md
| 复杂度 | 名称 | 示例 |
|--------|------|------|
| O(1) | 常数时间 | 数组下标访问 |
| O(log n) | 对数时间 | 二分查找 |
| O(n) | 线性时间 | 遍历数组 |
| O(n log n) | 线性对数 | 快速排序 |
| O(n²) | 平方时间 | 嵌套循环 |
| O(2ⁿ) | 指数时间 | 斐波那契递归 |

### 空间复杂度
- **O(1)**：原地操作，使用常数空间
- **O(n)**：使用与输入规模成比例的额外空间
- **O(n²)**：使用二维数组等
